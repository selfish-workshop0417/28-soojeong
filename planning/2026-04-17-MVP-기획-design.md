## 1. 제품 개요

### 한 줄 정의
씬스틸러는 영상 창작자가 레퍼런스를 "왜 훔쳤는지"와 함께 기록·공유하는 큐레이션 커뮤니티다.

### 핵심 가치
1. **아카이빙** — 흩어진 레퍼런스를 한곳에 모아 다시 찾기 쉽게
2. **이유 기록** — 왜 이 장면이 좋았는지 글로 남겨 내 안목을 훈련
3. **타인 엿보기** — 다른 창작자가 뭘 훔쳤는지 구경하며 영감 받기

### 차별점
| 경쟁 서비스 | 한계 | 씬스틸러의 차별 |
|---|---|---|
| 핀터레스트 | 이미지 중심, 저장 이유 없음 | 영상 중심 + 이유 필수 |
| 노션/북마크앱 | 혼자 쓰는 도구 | "공개 전제" 아카이브 |
| 유튜브 플레이리스트 | 단일 플랫폼, 맥락 없음 | 멀티 플랫폼 + 이유·태그 |

### 타겟 페르소나
20~30대 영상 창작자 (에디터·모션그래퍼·유튜버·CF 감독 지망생). 평소 인스타 저장·유튜브 "나중에 볼 영상"에 레퍼런스를 쌓지만, 정리도 안 되고 왜 저장했는지도 까먹는 사람.

### 지원 언어·지역
- 한국어 단일 (MVP)

## 2. 사용자 스토리

### 신규 방문자 (로그인 전)
- **US-01** 처음 들어온 사람으로서 / 로그인 없이 피드를 둘러보고 싶다 / 서비스 느낌을 먼저 감 잡도록
- **US-02** 신규 방문자로서 / 인기 있는 레퍼런스를 먼저 보고 싶다 / 콘텐츠 품질을 빠르게 판단하도록

### 가입 사용자 — 아카이빙
- **US-03** 창작자로서 / YouTube·Instagram 링크를 붙여넣어 저장하고 싶다 / 흩어진 레퍼런스를 한곳에 모으도록
- **US-04** 사용자로서 / 저장한 영상의 썸네일이 자동으로 뜨길 원한다 / 시각적으로 훑어보기 쉽도록
- **US-05** 사용자로서 / 내 영상들을 시간순으로 보고 싶다 / 과거 영감을 다시 찾도록

### 가입 사용자 — 이유 기록
- **US-06** 창작자로서 / 저장 시 "어떤 점이 좋았나 / 어디에 활용해보고 싶은가" 가이드 질문에 답하고 싶다 / 저장 맥락이 살아있도록
- **US-07** 창작자로서 / 이모지 태그를 클릭해 감정을 빠르게 남기고 싶다 / 글쓰기 부담 없이 기록하도록
- **US-08** 창작자로서 / 이유 텍스트의 공개·비공개를 선택하고 싶다 / 개인 프로젝트 정보를 보호하도록

### 가입 사용자 — 엿보기
- **US-09** 창작자로서 / 다른 사용자의 공개 아카이브를 구경하고 싶다 / 타인의 취향에서 영감을 얻도록
- **US-10** 사용자로서 / 인기 레퍼런스에 좋아요를 누르고 싶다 / 내 반응을 남기고 좋은 콘텐츠를 상위에 올리도록
- **US-11** 사용자로서 / 마음에 드는 크리에이터를 팔로우하고 싶다 / 새 저장물을 내 피드에서 보도록
- **US-12** 사용자로서 / 영상에 댓글로 반응하고 싶다 / 레퍼런스를 두고 대화하도록
- **US-13** 사용자로서 / 팔로우·좋아요·댓글 알림을 받고 싶다 / 재방문 이유가 생기도록

## 3. 기능 명세 (우선순위별)

### 🟢 P0 — 영감 카탈로그 단계
목표: 로그인 없이도 보는 재미가 있는 상태. 시드 콘텐츠로 서비스의 얼굴을 만든다.

| 기능 | 설명 |
|---|---|
| 피드 (읽기) | 카드형 그리드, 썸네일·제목·저장자·이모지 태그 노출 |
| 피드 정렬 | 최신순 / 인기순(좋아요 수) 탭 전환 |
| **비로그인 제한 UX** | 인기순 상단 8개만 선명, 하단부터 그라데이션 블러 + CTA "계속 훔쳐보려면? 1초 가입하고 열기" |
| 영상 상세 페이지 | 큰 썸네일 + 원본 링크 + 저장자 + 이유 텍스트 + 이모지 태그 + 좋아요 수 |
| 가벼운 가입/로그인 | 소셜 로그인 1개 (카카오 또는 Google), 닉네임 1회 입력 |
| 좋아요 | 로그인 상태에서 영상 카드·상세 페이지에 하트 버튼 |
| 운영자 시드 아카이브 | 팀+공모로 모은 크리에이터 5명 × 영상 10개씩 ≈ 50개 |

### 🟡 P1 — 내 서랍 단계
목표: 사용자가 직접 저장·기록·공개할 수 있는 상태.

| 기능 | 설명 |
|---|---|
| 링크 저장 | YouTube·Instagram URL 붙여넣기 → 썸네일·제목 자동 파싱 |
| 이유 작성 — 가이드 질문 | "어떤 점이 좋았나요?" / "어디에 활용해보고 싶으신가요?" 2칸 |
| 이유 작성 — 이모지 태그 | 미리 정의된 9개 중 1~3개 선택 |
| **이유 공개/비공개 토글** | 텍스트만 토글. 영상 카드·썸네일·태그·좋아요는 항상 공개. 기본값 공개 |
| 내 아카이브 페이지 | 내가 저장한 영상 시간순 그리드 |
| 공개 프로필 URL | `scenestealer.app/@username` 형식 |

### 🟠 P2 — 커뮤니티 단계
| 기능 | 설명 |
|---|---|
| 팔로우 | 크리에이터 팔로우 + "팔로잉 피드" 탭 |
| 댓글 | 영상 상세 페이지 스레드형 댓글 |
| 태그 기반 탐색 | 이모지 태그 클릭 시 해당 태그 필터 피드 |

### 🔵 P3 — 재방문 유도 단계
| 기능 | 설명 |
|---|---|
| 알림 센터 | 새 팔로워 / 내 영상 좋아요 / 댓글 인앱 알림 |

### ❌ MVP 제외 (의도적)
- 컬렉션/무드보드
- DM
- Vimeo·범용 URL 지원 (YouTube+Instagram만)
- 추천 알고리즘·개인화 피드
- 키워드 검색 (태그 필터로 대체)
- 모바일 네이티브 앱 (웹 우선)

## 4. 사용자 티어 (활성화 게이트)

단계적 잠금 해제로 "구경 → 가입 → 기여 → 커뮤니티" 여정 유도.

| 티어 | 조건 | 잠금 해제 |
|---|---|---|
| T0 | 비로그인 | 인기 상단 8개 미리보기 |
| T1 | 가입 | 전체 피드 열람 + 좋아요 |
| T2a | 좋아요 5개 누름 | 팔로우 |
| T2b | 저장 1개 이상 | 태그 필터 탐색 + 댓글 |

### 락 UX
- 팔로우/댓글/태그 필터 클릭 시 → 안내 모달 + 유도 버튼
  - 예: "첫 영상 1개만 훔쳐오면 열려요 🎬 [지금 훔치러 가기]"

## 5. 사용자 흐름

### Flow A — 신규 방문자 → 가입
```
랜딩 → 인기 피드 상단 8개 선명 노출
     → 스크롤: 블러 + "1초 가입하고 열기" CTA
     → 소셜 로그인 버튼 클릭
     → OAuth 동의 (카카오/Google)
     → 닉네임 1회 입력 (자동 @username 생성)
     → 전체 피드로 복귀
```

### Flow B — 영감 발견 → 좋아요 (P0 완결 루프)
```
피드 탐색 → 카드 클릭 → 영상 상세 (원본 링크·이유·태그·좋아요)
         → ♡ 좋아요 클릭
         → (P1 이후) 저장자 프로필 클릭 → 공개 아카이브 열람
```

### Flow C — 링크 저장 → 이유 기록 (P1 핵심 루프)
```
상단 "훔치기" 버튼 → URL 붙여넣기
                  → 썸네일·제목 자동 파싱 (실패 시 수동 입력)
                  → 가이드 질문 2개 답 작성
                     "어떤 점이 좋았나요?"
                     "어디에 활용해보고 싶으신가요?"
                  → 🔒 이 내용만 비공개 체크박스 (기본 해제)
                  → 이모지 태그 1~3개 선택
                  → [저장] → 내 아카이브 추가
                  → (선택) 공개 프로필 URL 복사해 공유
```

### Flow D — 재방문 유도 (P2~P3)
```
크리에이터 프로필 → [팔로우] (T2a 달성 후 가능)
                → 홈 "팔로잉" 탭 활성화
                → 그 사람이 새로 훔치면 내 피드에 표시
                → (P3) 좋아요·댓글·새 팔로워 알림
```

## 6. UI·톤 가이드

### 서비스 아이덴티티 워딩
- "훔치기" / "서랍" 등의 은어는 유지 (정체성)

### 입력 안내 톤
- 존댓말·부드러운 어조 (`~하고 싶으신가요?`, `~남겨보세요`)
- placeholder 예시:
  - "어떤 점이 좋았나요?" → *예: 전환 타이밍이 기가 막혀서…*
  - "어디에 활용해보고 싶으신가요?" → *예: 다음 브랜드 영상 인트로에*

### 이모지 태그 초기 세트 (9개, 추후 확장 가능)
🔥 충격 / 💡 아이디어 / 🎨 미감 / 🎬 연출 / ✨ 무드 / 😮‍💨 여운 / 🕊 미니멀 / ⚡ 속도감 / 🎭 감정

## 7. 성공 지표 (KPI)

### P0 지표
| 지표 | 목표 |
|---|---|
| 방문자 → 가입 전환율 | 15% 이상 |
| 첫 방문 체류 시간 | 평균 90초 이상 |
| 좋아요 누른 사용자 비율 | 가입자의 40% 이상 |
| 재방문율 (D7) | 20% 이상 |

### P1 지표
| 지표 | 목표 |
|---|---|
| 첫 저장 전환율 (T1→T2b) | 50% 이상 |
| 이유 작성률 | 저장 건의 70% 이상 |
| 평균 저장 수 / 활성 사용자 | 주 3개 이상 |
| 공개 프로필 공유 횟수 | 가입자의 10% 이상 |

### P2 지표
| 지표 | 목표 |
|---|---|
| 팔로우 관계 수 / 사용자 | 평균 5명 이상 |
| 댓글 작성률 | 영상 10개당 1개 이상 |
| 태그 기반 탐색 사용률 | 세션의 20% 이상 |

### P3 지표
| 지표 | 목표 |
|---|---|
| 알림 클릭율 | 30% 이상 |
| D30 리텐션 | 가입자의 15% 이상 |

### 🚨 안티 지표 (나쁜 신호)
- 저장 후 이유 작성률 < 50% → 입력 UX 재설계
- 피드 첫 스크롤 전 이탈률 > 70% → 시드 콘텐츠 품질 문제
- 이유 비공개 전환 비율 > 50% → "공개 아카이브" 전제 흔들림
- T1→T2b 전환율 < 50% → 저장 허들 과도

## 8. 리스크 & 대응

| # | 리스크 | 증상 | 완화책 |
|---|---|---|---|
| R1 | 콜드 스타트 실패 | 시드 50개 매력 없음 → 가입 전환 저조 | 크리에이터 공모로 시드 확충, 팀이 "훔친 이유"까지 진짜 톤으로 작성 |
| R2 | 저작권·초상권 | 원작자 이의 제기 | 원본 링크·출처 명시, 썸네일+링크만 사용(영상 호스팅 X), 신고 시 즉시 삭제 프로세스 |
| R3 | "훔친다" 워딩 거부감 | 부정적 뉘앙스 오해 | 서비스 스토리 페이지에 "왜 훔친다고 하는가" 설명 (창작자 은어·창작 본질) |
| R4 | T2b 허들 과도 | 커뮤니티 기능 거의 안 쓰임 | 저장 UX 극단적으로 가볍게(URL 붙여넣기 1스텝), 가이드 질문 스킵 허용 검토 |
| R5 | 비공개 남용 | 엿보기 가치 붕괴 | 비공개 비율 모니터링, 50% 초과 시 UX 개선 |
| R6 | 민감 콘텐츠 유입 | 성인물·저작권 침해 영상 게시 | 필터링 정책 수립 및 신고·검토 프로세스 운영 |
